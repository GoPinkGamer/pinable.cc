快速开始

1. 下载并安装 PinableAgents 桌面端（macOS / Windows）。

2. 打开应用，在 Modules 页面启用 do、essentials 等基础模块。

3. 在 Workflows 页面选择 Codex / Claude / Gemini 后端，输入任务描述。

4. 首次运行建议使用 do 工作流，确保需求澄清与实施闭环。

工作流对比

do（推荐）：7 阶段交付流程，适合大多数功能需求。

omo：风险信号驱动路由，适合定位未知问题与复杂重构。

bmad：企业级敏捷流程，适合多角色协作与大型项目。

sparv：轻量 5 阶段流程，适合快速原型与小型任务。

模块与配置

模块开关统一由 config.json 控制，建议先启用 do + essentials。

冲突策略：overwrite（覆盖）、skip（跳过）、backup（备份后覆盖）。

典型组合：功能交付（do + essentials）、Bug 修复（omo + develop）。

多后端执行

支持 Codex / Claude / Gemini。通过统一 CLI 参数保持一致体验。

会话恢复：执行失败后可通过 session id 继续任务。

建议：不同后端适配不同角色，例如 Claude 做规划，Codex 做实现。

pinable-desktop 使用说明

Modules：安装/卸载模块，支持批量选择与进度可视化。

Workflows：选择后端、输入任务描述、配置工作目录并实时查看输出。

Config：编辑 config.json，支持校验与回滚。

常见问题

安装失败：检查是否有写入权限，或重新安装模块。

后端 CLI 缺失：确认 codex/claude/gemini 在 PATH 中。

权限提示：建议在团队策略允许的情况下配置自动审批。
